# Storage Service

S3-compatible object storage client for AWS S3, DigitalOcean Spaces, MinIO, and more.

## Architecture Pattern

This module promotes the **Pre-signed URL** pattern. 
- **Upload**: Backend generates a PUT URL -> Frontend uploads directly to S3.
- **Download**: Backend generates a GET URL -> Frontend/User downloads directly from S3.

This ensures your server never handles binary data streams, drastically reducing CPU and bandwidth costs.

## Configuration

Required environment variables:
- `S3_ENDPOINT`: e.g., `https://nyc3.digitaloceanspaces.com`
- `S3_REGION`: e.g., `nyc3`
- `S3_BUCKET`: Your bucket name
- `S3_ACCESS_KEY`
- `S3_SECRET_KEY`

## Usage

### Generate Download URL

```typescript
import { createStorageClientFromEnv } from '@aptly/services/storage';

const storage = createStorageClientFromEnv();

// Generates a public URL valid for 1 hour by default
const url = await storage.getDownloadUrl('invoices/123.pdf');
```

### Direct Uploads from Frontend

```typescript
// Backend: Generate the target URL
const url = await storage.getUploadUrl('uploads/photo.jpg', 'image/jpeg');

// Frontend: Call the PUT URL directly
await fetch(url, { method: 'PUT', body: fileFileObject });
```

## AI Agent Integration Note

When creating file paths, always use a hierarchical structure (e.g., `tenant_id/category/item_id.ext`). The `storage` client does not enforce folder structures, so you must define them in your application logic.
